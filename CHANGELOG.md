# Changelog

## [0.1.2]
- Улучшен сценарий Deploy:
	- если `.fboot` отсутствует, показывается диалог с вариантами `Создать` / `Отмена`;
	- при выборе `Создать` выполняется генерация `.fboot`, затем продолжается Deploy.
- Улучшен сценарий `Создать FBOOT`:
	- если `.fboot` уже существует, показывается подтверждение `Перезаписать` / `Отмена`.
- Исправлено разрешение путей библиотек:
	- `Type Library` теперь обрабатывается как локальный путь текущего `.sys`;
	- исключено «перетекание» `Type Library` между разными проектами.
- После создания нового FB type автоматически обновляется панель `Библиотека блоков`, если она открыта.
- После сохранения настроек (изменения путей) автоматически обновляется панель `Библиотека блоков`, если она открыта.
- Исправлено отображение `Класс` у новых блоков, добавленных с помощью drag-and-drop из библиотеки.
- Добавлена модель языков алгоритмов (`AlgorithmLanguage`) с централизованным маппингом язык -> XML-тег при сериализации (`ST` -> `<ST>`, `Python` -> `<PY>`).

## [0.1.1]
- Добавлена функциональность создания FB типов: `Basic FB` и `Simple FB`.
- Улучшено начальное отображение диаграмм: адаптивное масштабирование и авто-fit для схем разного размера.
- Добавлено отображение значений параметров у портов на canvas.
- Для входных data-портов добавлены дефолтные значения по типам (включая BOOL и TIME/DATE типы), если параметр не задан.

## [0.1.0]
- Диалог настроек плагина с сохранением параметров.
- Редактирование параметров функциональных блоков и OPC mapping.
- Удаление блоков, удаление и выделение соединений.
- Сохранение изменений диаграммы обратно в `.sys`.
- Генерация `.fboot` и deploy в OpenFB runtime.


## [0.0.1]
- Первоначальный релиз.
- Визуальный рендеринг диаграмм.
- Интерактивный canvas.
- Отладочное логирование.
