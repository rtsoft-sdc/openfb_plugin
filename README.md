# OpenFB Plugin для VS Code

OpenFB Plugin добавляет в VS Code интерактивный редактор диаграмм IEC 61499 Function Block (FBD) для проектов в 
формете 4diac и forgelogic (файлов `.sys`). Позволяет работать со средой исполнения OpenFB.

## Что умеет

- Открывает и визуализирует проект IEC 61499 в виде файла`.sys`, как интерактивную диаграмму
- Поддерживает pan/zoom, выделение и drag-and-drop блоков
- Создаёт связи между портами (event/data) с валидацией совместимости
- Удаляет блоки и связи (`Delete` и контекстное меню)
- Редактирует параметры блоков и OPCUA mapping (помечает входы и выходы, как публикуемые через OPCUA сервер, используя 
формат совместимый с forgelogic)
- Сохраняет изменения проекта обратно в `.sys`
- Генерирует `.fboot` и выполняет загрузку (deploy) в среду исполнения OpenFB
- Показывает логи в webview и extension output
- При загрузке проекта в среду исполнения (deploy) предлагает создать `.fboot`, если файл отсутствует
- При генерации `.fboot` спрашивает подтверждение на перезапись существующих файлов
- Автоматически обновляет панель `Библиотека блоков`:
	- после создания нового FB type
	- после сохранения настроек путей библиотек
- Текущая версия позволяет использовать только один вычислительный узел в аппаратной конфигурации

## Быстрый старт

1. Установите расширение в VS Code.
2. В Explorer нажмите правой кнопкой на `.sys` файле.
3. Выберите команду **OpenFB: Открыть диаграмму проекта**.


## Настройки

- `openfb.fbLibraryPaths` — пути поиска библиотек `.fbt`
- `openfb.host` — адрес OpenFB runtime
- `openfb.port` — порт OpenFB runtime
- `openfb.deployTimeoutMs` — таймаут deploy в миллисекундах

## Последние изменения

- Обновлены диалоги генерации/загрузки `.fboot` с подтверждением действий.
- Сообщения о генерации и загрузки показывают имена файлов без полного пути.
- Исправлено отображение класса у новых блоков из библиотеки.
- Улучшена обработка путей `Type Library` для нескольких `.sys` проектов.
